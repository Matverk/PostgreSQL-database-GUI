## Программа с графическим интерфейсом для работы с базой данных "Автобусные маршруты Зеленограда" в PostgreSQL

Супер простое взаимодействие как с пользователем, так и с базой данных.

На вкладке *"Вставить строки"* можно выбрать таблицу **для вставки новой строки**, а потом заполнить значения атрибутов. Значения `""` (пустая строка), `"None"` и `"null"` трактуются как NULL при заполнении БД.

На вкладке *"Выбрать строки"* можно выбрать таблицу **для просмотра** всех находящихся в ней **строк** и отсортировать их по возрастанию значений одного из атрибутов.

Для того, чтобы **изменить** значения столбцов какой-либо строки, нужно сделать двойное нажатие `<левой кнопкой мыши>` по этой строке. Откроется вкладка *"Изменить строки"*. На ней можно ввести новые значения атрибутов этой строки и отправить запрос на обновление серверу БД с помощью кнопки "Обновить строку".

Для того, чтобы **удалить** одну или несколько строк из выведенной таблицы, нужно выбрать эти строки `<ЛКМ>` с зажатой клавишей `<Ctrl>` или `<Shift>`, а затем нажать `<правую кнопку мыши>`. Откроется вкладка *"Удалить строки"*. На ней будут отображены строки, выбранные для удаления. Подтвердить удаление можно с помощью кнопки "Удалить".

В нижней части окна программы находится статусная строка, кратко отображающая результат выполнения действий пользователя. Над ней по разным сторонам окна расположены 2 кнопки:

- "Откатить" - возвращает состояние просматриваемой БД в исходное
- "Зафиксировать" - делает внесённые изменения видимыми для всех пользователей БД

#### ❗ При возникновении ошибки при выполнении запроса к серверу БД (неправильный ввод пользователя или проблема с сервером) выполняется отмена всех изменений, внесённых после момента последней фиксации (нажатия кнопки "Зафиксировать").

Такое поведение сделано для намеренно. Для предотвращения неприятных ситуаций рекомендуется фиксировать изменения после каждой успешной операции изменения содержимого таблиц в базе данных.
